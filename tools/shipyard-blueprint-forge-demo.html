📋 System Overview

The Shipyard Blueprint Forge enables players to design custom spacecraft by selecting hulls, assigning modules to slots, and previewing real-time performance statistics. Built on the isometric module visualization system, it provides an intuitive game-quality interface for ship configuration with full API integration.

  • Hull selection with class-based organization
  • Slot-based module assignment with visual previews
  • Real-time performance statistics calculation
  • Blueprint save/load/export workflows
  • Ship building from completed blueprints
  • Faction affinity tracking and tech gating
  • Loadout preset system with localStorage persistence
  • Interactive module selection modals

💻 Usage Example

// Initialize the Forge const forge = new ShipyardBlueprintForge({ container: document.getElementById('shipyard'), colonyId: 42, apiBase: '/api' }); // Load data and render UI await forge.init(); // User selects hull and assigns modules via UI... // Save blueprint const result = await forge.saveBlueprint('Fighter Fleet Leader'); // Build ship const ship = await forge.buildShip(); console.log('Built ship:', ship.vessel_id); // Listen for stats changes window.addEventListener('gq:blueprint-stats-changed', ev => { console.log('Stats updated:', ev.detail); });

🏗️ Architecture

The Forge follows a modular architecture with clear separation of concerns:

Component Responsibility File
ShipyardBlueprintForge Main controller, data management js/ui/ShipyardBlueprintForge.js
IsometricModuleRenderer Canvas-based module visualization js/ui/IsometricModuleRenderer.js
CSS Styling UI theme and component styles css/isometric-modules.css
Shipyard API Backend endpoint handlers api/shipyard.php
Database Schema Persistent data storage sql/migrations
